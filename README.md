# 💌 Приглашение на свидание

Романтичный одностраничный сайт-приглашение. Тёмный бордовый фон с золотыми акцентами,
плавающие сердечки, кнопка «Нет», которая невозможна к нажатию, и финальный экран-билет
с местом/датой/временем.

Ответ девушки сохраняется в Firebase Firestore — это значит, что **по той же самой ссылке**
вы оба видите результат: она отвечает один раз, а вы, открыв ссылку позже, сразу видите
её ответ (без email и без Telegram).

## Файлы
- `index.html` — структура
- `style.css` — стили
- `script.js` — логика + интеграция с Firestore

## Публикация на GitHub Pages
1. Создайте репозиторий на GitHub (или используйте существующий).
2. Загрузите туда три файла: `index.html`, `style.css`, `script.js`.
3. Settings → Pages.
4. В разделе Source выберите:
   - Branch: `main`
   - Folder: `/ (root)`
5. Нажмите Save.
6. Через 1–2 минуты сайт появится по адресу:
   `https://<ваш-username>.github.io/<репозиторий>/`

## Как это работает с Firestore
- При первом открытии страницы в адресной строке появляется `?id=...` — уникальный
  идентификатор этого приглашения.
- Когда девушка отвечает (выбирает место/дату/время и жмёт «Отправить»), данные
  записываются в Firestore в документ с этим `id`.
- Если открыть **ту же самую ссылку** ещё раз (например, чтобы посмотреть ответ) —
  сайт сразу показывает финальный экран с результатом, минуя вопросы.
- Поэтому ссылку, которую вы отправляете девушке, стоит сохранить себе — именно
  по ней вы потом увидите ответ.

## Правила доступа Firestore
В консоли Firebase: Firestore Database → вкладка Rules — должно быть открыто
чтение/запись (test mode):

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}
```

⚠️ Это открытый доступ — подходит для личного небольшого проекта,
но не для публичного продукта с большим количеством пользователей.
